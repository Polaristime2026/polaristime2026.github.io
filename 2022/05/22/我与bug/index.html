我与bug

说来 bug 这种东西，对于学习计算机的我们来说。已经习以为常啦，好像在一个项目里面没有几个 bug 都觉得。
这个项目不是那么的“完美”！对于 bug 我的态度是既来之则安之！

PySimpleGUI

我看了一下 知乎上对于 PySimpleGUI 的定义写的非常复杂，我给各位复制一段感受一下，
红色 PySimpleGUI库是2018年发布的个人开源维护项目，是基于Tkinter、Remi、wxpython、PyQt这些模块的,其中Remi模块是将应用程序的界面用HTML的形式来展现，所以这个包有个亮点就是你可以输入相应的代码将其转变网页形式展示，而其余包则都是用来做图形界面的。
当时我自己看完上面这段话，头都大了。对于新手玩家来说，不是要去面试去入行工作的人来说，我觉得一句话概括一下就好，
PySimpleGUI 是用于 python 的开源的GUI第三方库。

我为什么会用到PySimpleGUI

其实是因为我的一个姐姐要给她的学生们，讲 PySimpleGUI 但是她需要实战案例，于是找上了我.从这里开始我的噩梦就来临了
众所周知py的第三方库非常丰富，这是它的优点也可以说是它的缺点，优点在于很多库只需要 pip install 就能用。缺点在于Python的版本更新非常快，有些比较老的库。高版并不支持，最后也是我个人感觉python在中最麻烦的一点pip的安装

pip的安装

pip 的安装，对于我个人来说是非常烦的，首先安装大部分的库你都是要换源的，就拿我这次安装 PySimpleGUI 来举例子

如上图所示真的非常麻烦，除了图片上的一个错误之外还有一大推的其他错误因为没有截图，我就不细说了，最后我是直接换源解决了上面的问题所以遇到pip问题最直接的方式就是换源，升级！
